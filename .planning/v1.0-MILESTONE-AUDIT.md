## Gaps — Undefined CSS Custom Properties

All 4 gaps are the same category: CSS custom properties referenced in component modules but never defined in `styles/global.css`. These were likely present in the original SCSS partials (`_colors.scss`, `_variables.scss`) but missed during the Phase 2 extraction.

### 1. `--purple-cactus-flower` (CSS-02, VER-07)
- **Used in:** `components/footer/footer.module.css:204`
- **Purpose:** Footer signature link hover color
- **Impact:** Hover state produces no visible color change
- **Fix:** Add `--purple-cactus-flower: <value>;` to `:root` in `styles/global.css`

### 2. `--laurens-lace` (CSS-02, VER-02)
- **Used in:** `components/header/header.module.css:107`
- **Purpose:** Mobile nav item text color
- **Impact:** Mobile nav text uses inherited default instead of brand color
- **Fix:** Add `--laurens-lace: <value>;` to `:root` in `styles/global.css`

### 3. `--padding-x` (CSS-03)
- **Used in:** `components/header/header.module.css:165`
- **Purpose:** Navigation categories padding
- **Impact:** Categories padding defaults to 0
- **Fix:** Add `--padding-x: <value>;` to `:root` in `styles/global.css`

### 4. `--z-content` (CSS-03, CSS-10)
- **Used in:** `components/wrapper/wrapper.module.css:14`
- **Purpose:** Main content area z-index stacking
- **Impact:** z-index defaults to `auto` (likely acceptable but not intentional)
- **Fix:** Add `--z-content: <value>;` to Section 5 of `styles/global.css`
