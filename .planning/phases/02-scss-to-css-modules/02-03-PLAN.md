Apply the same Tailwind-first pattern dictionary from Plan 02:

**vw() replacement -- Tailwind classes are PRIMARY (per user decision):**
- For each `desktop-vw(Npx)` call: identify the CSS property, find the Tailwind equivalent class, remove the line from CSS, add the Tailwind class to the corresponding JSX element.
- Tailwind mapping: font-size -> `text-*`, padding -> `p-*`/`px-*`/`py-*`, margin -> `m-*`, gap -> `gap-*`, width -> `w-*`, height -> `h-*`, border-radius -> `rounded-*`. Use arbitrary values `text-[22px]`, `p-[10px]` when no standard step exists.
- For `mobile-vw(Npx)` inside `@media (max-width: 800px)` blocks: use fixed rem/px values in CSS or `calc(N / 375 * 100vw)` only when fluid mobile scaling is essential.
- **FALLBACK (calc):** Use `calc(N / 1440 * 100vw)` ONLY for properties without Tailwind equivalents (clip-path, background-position, complex transforms, CSS custom property definitions).

**When adding Tailwind classes to JSX:**
- For each vw() call removed from CSS, add the Tailwind class to the JSX element using that CSS module class.
- Use the project's `cn()` utility or template literal to combine CSS module class with Tailwind classes.

**Mixin replacements (stay in CSS modules):**
- `@include mobile { ... }` -> `@media (max-width: 800px) { ... }`
- `@include desktop { ... }` -> `@media (min-width: 800px) { ... }`
- `@include hover { ... }` -> `@media (hover: hover) { ... }`
- `@include dims($w, $h)` -> `width: $w; height: $h;`
- `@include position(...)` -> expanded position properties
- `z-index('name')` -> `var(--z-name)`
- SCSS `//` comments -> `/* */` or remove
- Native CSS nesting: `& ` prefix for element selectors
