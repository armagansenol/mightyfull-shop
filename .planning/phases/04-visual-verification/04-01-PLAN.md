    Before anything else, start the dev server (`pnpm dev` in background), resize browser to 1440x900, navigate to http://localhost:3000, and verify that `md:` and `xl:` Tailwind responsive classes produce actual CSS rules. The homepage intro section uses `flex md:grid grid-cols-12` — if it renders as a multi-column grid on desktop, responsive variants are working. If it renders as a single-column vertical stack, responsive variants are BROKEN and must be fixed first.

    To diagnose if broken: use `browser_evaluate` to run `getComputedStyle` on an element with `md:grid` class. If `display` is not `grid` at 1440px, the `@source` directive in `styles/global.css` is not correctly covering the TSX files. Fix by adjusting the `@source` path to ensure it covers `app/` and `components/` directories. Re-verify after fix.

    4. Take a screenshot of dev build at same scroll positions
    5. Compare: layout structure, colors, typography, spacing, element presence
    6. If differences found: fix CSS module or TSX file immediately, wait for HMR, re-verify the page
    7. Move to next page
