  <action>
These files use `desktop-vw()`, `mobile-vw()`, `@include mobile`, `@include hover`, and `@include reduced-motion`. Apply these conversion patterns:

**Pattern dictionary (apply to ALL files) -- Tailwind-first per user decision:**

**vw() replacement -- Tailwind classes are the PRIMARY approach:**
- For each `desktop-vw(Npx)` call, determine the CSS property and convert to Tailwind:
  - **font-size:** `desktop-vw(16px)` -> remove from CSS, add `text-base` to JSX element. `desktop-vw(20px)` -> `text-xl`. `desktop-vw(24px)` -> `text-2xl`. `desktop-vw(14px)` -> `text-sm`. `desktop-vw(12px)` -> `text-xs`. For values without a Tailwind step, use arbitrary: `desktop-vw(22px)` -> `text-[22px]`.
  - **padding:** `desktop-vw(8px)` -> `p-2`. `desktop-vw(16px)` -> `p-4`. `desktop-vw(24px)` -> `p-6`. `desktop-vw(32px)` -> `p-8`. For directional: `px-*`, `py-*`, `pt-*`, `pb-*`, `pl-*`, `pr-*`. For values without a step: `p-[22px]`.
  - **margin:** Same as padding but `m-*` variants.
  - **gap:** `desktop-vw(8px)` -> `gap-2`. `desktop-vw(16px)` -> `gap-4`. Arbitrary: `gap-[22px]`.
  - **width:** `desktop-vw(100px)` -> `w-[100px]`. `desktop-vw(50%)` stays as CSS. Fixed sizes: `w-2` (8px), `w-4` (16px), `w-8` (32px), etc.
  - **height:** Same as width but `h-*`.
  - **border-radius:** `desktop-vw(8px)` -> `rounded-lg`. `desktop-vw(4px)` -> `rounded`. `desktop-vw(16px)` -> `rounded-2xl`. Arbitrary: `rounded-[10px]`.
- For `mobile-vw(Npx)` inside `@include mobile` blocks: the responsive breakpoint behavior is handled by `@media (max-width: 800px)` in the CSS module. Inside that media query, replace `mobile-vw(Npx)` with the equivalent Tailwind value in the CSS (e.g. `font-size: 0.875rem` for 14px) OR keep as `calc(N / 375 * 100vw)` ONLY when fluid mobile scaling is needed.
- **FALLBACK (calc) -- use ONLY when no Tailwind equivalent exists:** For CSS properties that Tailwind cannot express (e.g. `clip-path`, `background-position`, complex `transform` values, CSS custom property definitions in :root), use `calc(N / 1440 * 100vw)` as inline CSS in the module.

**When adding Tailwind classes to JSX:**
- For each vw() call removed from CSS, add the corresponding Tailwind class to the JSX element that uses the CSS module class.
- Example: if `.card` had `font-size: desktop-vw(20px)`, remove that line from CSS, and in JSX change `className={s.card}` to `className={cn(s.card, "text-xl")}` (using the project's `cn()` utility or template literal).
- If the element already has a className expression, append the Tailwind class.

**Mixin replacements (these stay in CSS modules, NOT Tailwind):**
- `@include mobile { ... }` -> `@media (max-width: 800px) { ... }` (native CSS nesting inside the selector).
- `@include desktop { ... }` -> `@media (min-width: 800px) { ... }`.
- `@include hover { ... }` -> `@media (hover: hover) { ... }`.
- `@include reduced-motion { ... }` -> `@media (prefers-reduced-motion: reduce) { ... }`.
- `@include dims($w, $h)` -> `width: $w; height: $h;` (expand the two values).
- `@include dims($s)` -> `width: $s; height: $s;` (single arg = square).
- `@include position($pos, $t, $r, $b, $l)` -> `position: $pos; top: $t; right: $r; bottom: $b; left: $l;`.
- `z-index('name')` -> `var(--z-name)`.
- SCSS `//` comments -> `/* */` or remove.
- SCSS `$variable: value` local variables -> replace with literal values or CSS custom properties.

**Per-file notes:**

1. **`marquee/marquee.module.scss`** -- Uses `@include mobile` and `@include reduced-motion`. No vw() calls. Straightforward media query replacement. No JSX Tailwind changes needed.

2. **`noticebar/noticebar.module.scss`** -- 6 vw() calls, 2 `@include mobile`. For each vw() call: identify the property (font-size, padding, etc.), remove from CSS, add Tailwind class to JSX. Update the component TSX file alongside the CSS module.

3. **`quantity/quantity.module.scss`** -- 6 vw() calls, 2 `@include mobile`. Same Tailwind-first approach. Update component TSX.

4. **`product-card/product-card.module.scss`** -- 7 vw() calls, 3 `@include mobile/desktop`. Replace vw() with Tailwind classes in JSX. Watch for nested selectors with element names -- add `&` prefix in CSS.

5. **`scrollbar/scrollbar.module.scss`** -- **PITFALL:** Line ~23 uses `@media (--mobile)` which is a CSS custom media query (not implemented in browsers). Replace with `@media (max-width: 800px)`. 3 vw() calls -> Tailwind classes in JSX.

6. **`cart/edit-quantity-button/edit-quantity-button.module.scss`** -- 3 vw() calls, no @include mobile. Replace vw() with Tailwind classes in JSX.

**For each file:**
1. Read the SCSS file
2. For each vw() call: identify the property, find the Tailwind equivalent, note it
3. Apply mixin replacements in CSS (media queries, dims, position)
4. Remove vw()-based property lines from CSS where Tailwind replaces them
5. Write as .module.css, delete the .module.scss
6. Open the corresponding JSX/TSX file
7. Add Tailwind classes to elements that use the affected CSS module classes
8. Update the JSX import path from .module.scss to .module.css

After all 6, run `pnpm build` to verify.
