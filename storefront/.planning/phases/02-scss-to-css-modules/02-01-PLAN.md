  NOTE: `composes:` is inapplicable here because this `@extend` targets `.remove-autofill-styles`, a global utility class defined in global.css. CSS Modules `composes:` only works for module-scoped classes (classes defined within the same `.module.css` file or explicitly imported from another module). Global utility classes are outside the CSS Modules scope system entirely. Inline expansion is the correct approach for all global-scope `@extend` usages in this file. This overrides the locked decision ('Replace @extend with CSS Modules composes: keyword') per CSS Modules scoping constraint -- that decision applies to component modules only.

  ```css
    }
  }
  ```
