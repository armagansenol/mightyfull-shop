**Step 1: Convert `styles/buttons.module.scss` to `styles/buttons.module.css`**

The file has 7 button variants plus 3 size classes. Apply these conversions:

- `.default` -- no SCSS syntax, copy as-is.
- `.blue-ruin`, `.inverted-blue-ruin`, `.themed`, `.inverted-themed` -- each has `@include hover { &:hover { ... } }`. Replace with:
  ```css
  @media (hover: hover) {
    &:hover { ... }
  }
  ```
  Note: the `&.hover-animation` compound selector works in native CSS nesting. The structure becomes:
  ```css
  .blue-ruin {
    /* ... base styles ... */
    &.hover-animation {
      @media (hover: hover) {
        &:hover {
          /* ... hover styles ... */
        }
      }
    }
  }
  ```
- `.naked-blue-ruin`, `.naked-themed`, `.naked-full` -- no SCSS syntax, copy as-is.
- `.sm` -- OMIT `font-size` entirely from this class (see JSX step below). Keep `@include mobile` conversion for any non-font-size rules using `@media (max-width: 800px)`.
- `.md` -- OMIT `font-size` entirely from this class (see JSX step below). Keep `@include mobile` conversion for any non-font-size rules.
- `.lg` -- OMIT `font-size` entirely from this class (see JSX step below). Keep `@include mobile` conversion for any non-font-size rules.

  RATIONALE: Per the locked decision in CONTEXT.md ('replace vw() with Tailwind responsive classes -- apply to everything possible: font sizes'), font sizes from `desktop-vw()` must become Tailwind `text-*` classes in JSX, NOT `calc()` expressions in CSS. Converting to `calc()` here would contradict the locked decision.

  Font-size Tailwind mappings for button size variants (desktop-vw source → nearest Tailwind class):
  - `.sm`: `desktop-vw(16px)` → `text-base` (16px base)
  - `.md`: `desktop-vw(24px)` → `text-2xl` (24px base)
  - `.lg`: `desktop-vw(30px)` → `text-3xl` (30px base)

Write as `styles/buttons.module.css`. Delete `styles/buttons.module.scss`.

**Step 2: Update import and add Tailwind size classes in `components/ui/button.tsx`**

Change: `import s from '@/styles/buttons.module.scss'` (or similar path)
To: `import s from '@/styles/buttons.module.css'`

Additionally, add Tailwind `text-*` classes to the JSX wherever the `sm`, `md`, or `lg` size class is applied. Because `font-size` is NOT in the CSS module (it was removed per the locked decision), font size must come from Tailwind classes in JSX:
- When size is `sm`: add `text-base` to the element's className alongside `s.sm`
- When size is `md`: add `text-2xl` to the element's className alongside `s.md`
- When size is `lg`: add `text-3xl` to the element's className alongside `s.lg`

Inspect `components/ui/button.tsx` to find the exact conditional logic that applies size variant class names, then add the corresponding Tailwind class in the same expression. Example pattern:
```tsx
// Before (size applied as CSS module class only):
className={cn(s[variant], s[size])}

// After (size class retains CSS module for non-font-size rules; Tailwind handles font-size):
className={cn(s[variant], s[size], {
  'text-base': size === 'sm',
  'text-2xl': size === 'md',
  'text-3xl': size === 'lg',
})}
```
Adapt the exact implementation to match the real structure found in button.tsx.
