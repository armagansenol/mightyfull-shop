<task type="auto">
  <name>Task 1: Migrate from npm to pnpm</name>
  <files>package.json, package-lock.json, pnpm-lock.yaml, .npmrc</files>
  <action>
Step 1: Enable pnpm via corepack:
```bash
corepack enable pnpm
```

Step 2: Delete npm lock file:
```bash
rm package-lock.json
```

Step 3: Add pnpm peer dependency config to package.json to suppress the Hydrogen React vite peer dep warning (per research — vite is optional for non-Vite consumers):
```json
{
  "pnpm": {
    "peerDependencyRules": {
      "ignoreMissing": ["vite"]
    }
  }
}
```

Step 4: Run pnpm install to generate pnpm-lock.yaml:
```bash
pnpm install
```

If MODULE_NOT_FOUND errors appear during install, create `.npmrc` with `shamefully-hoist=true` as an escape hatch (per user decision: use only if specific compatibility issues arise). Otherwise, do NOT create `.npmrc` — use pnpm's default content-addressable structure.

Step 5: Update package.json scripts — no changes needed since `next dev`, `next build`, `next start` are package-manager agnostic. The `lint` script will be updated in Plan 01-03 when Biome replaces ESLint.

Step 6: Verify the build works with pnpm:
```bash
pnpm build
```

Step 7: Verify dev server starts:
```bash
pnpm dev
```
Start it, confirm it starts without errors, then stop it.

Step 8: Commit this as a working state:
```
git add package.json pnpm-lock.yaml .npmrc 2>/dev/null
git rm package-lock.json 2>/dev/null
git commit -m "chore(01-01): migrate from npm to pnpm"
```
  </action>

  </verify>
  <done>pnpm-lock.yaml exists, package-lock.json is deleted, pnpm build succeeds, dev server starts without errors</done>
</task>

<task type="auto">
  <name>Task 2: Verify Next.js and React are current, confirm async API compliance</name>
  <files>package.json</files>
  <action>
Step 1: Verify Next.js and React are already at latest stable. Run:
```bash
pnpm dlx npm-check-updates --filter "next,react,react-dom" --target latest
```
Confirm no updates needed (Next.js 16.1.6, React 19.2.4 are current per research).

Step 2: Run the Next.js codemod in dry mode to confirm nothing needs changing:
```bash
pnpm dlx @next/codemod@canary upgrade latest --dry
```
The codemod should report no changes needed since the project is already on 16.1.6 with async APIs migrated. If the codemod tries to modify anything, apply the changes and review.

Step 3: Spot-check async API compliance. Verify these files use the await pattern correctly:
- `app/(main)/shop/[slug]/page.tsx` — params should be `Promise<{slug}>` with `await params`
- `app/layout.tsx` — `await cookies()` pattern
- `lib/shopify/index.ts` — `await cookies()` pattern
- `components/cart/actions.ts` — `await cookies()` pattern

Grep for any remaining sync params or cookies usage:
```bash
grep -rn "cookies()" --include="*.ts" --include="*.tsx" app/ lib/ components/ | grep -v "await cookies"
grep -rn "params\." --include="*.tsx" app/ | grep -v "await params" | grep -v "searchParams" | grep -v "node_modules"
```

If any non-awaited usage is found, fix it using the async pattern.

Step 4: Check for middleware.ts that might need renaming to proxy.ts per Next.js 16. Research confirmed no project middleware.ts exists (only in node_modules), so this should be a no-op.

Step 5: Verify build still passes after any changes:
```bash
pnpm build
```

Commit if any fixes were made:

