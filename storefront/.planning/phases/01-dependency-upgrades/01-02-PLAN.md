Major version jumps but research indicates low risk (only breaking change is React 19.2 requirement, already met).

```bash

```

Target versions:
- sanity: 3.62.0 -> 5.12.0
- next-sanity: 9.12.3 -> 12.1.0
- @portabletext/react: 3.1.0 -> 6.0.2

Step 5: After upgrade, verify these import patterns still work:
- Check `app/api/revalidate/route.ts` — `parseBody` from `next-sanity/webhook` (verify import path unchanged)

- Check `components/customized-portable-text/CustomizedPortableText.tsx` — imports from `@portabletext/react`
- Check `components/faq-list/index.tsx` — imports from `@portabletext/react`

Fix any TypeScript errors from changed APIs. Common issues:
